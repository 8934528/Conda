# Contributing to Cobra IDE

Thank you for your interest in contributing to Cobra IDE! I welcome contributions from the community to help make this the best game development environment.

## How to Contribute

### Reporting Bugs

If you find a bug, please report it by creating an issue. Include:

- A clear, descriptive title.
- Steps to reproduce the bug.
- Expected vs. actual behavior.
- Screenshots or recordings if applicable.

### Suggesting Features

I love new ideas! To suggest a feature:

- Check if the feature has already been requested.
- Provide a clear explanation of how the feature would work and its benefits.

### Pull Requests

1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Ensure your code follows the project's coding standards.
4. Submit a pull request with a detailed description of your changes.

## Coding Standards

- Use consistent indentation (4 spaces).
- Follow C# naming conventions (PascalCase for classes and methods).
- Add comments and docstrings where necessary to explain complex logic.
- Ensure all new UI components follow the **Dark Orange** theme (#ff8c00).

## UI/UX Guidelines

- Use `Grid` and `GridSplitter` for resizable panels.
- Standardize on `FontSize="12"` for text elements.
- Use `MahApps.Metro.IconPacks` for icons (standard size `14x14`).
- Maintain consistent spacing between toolbar elements.

Thank you for helping us improve Cobra IDE!
